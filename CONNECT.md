# Подключение нод (P2P)

## 1. Одиночная нода

```bash
./waters-node --port 42069
# → http://localhost:42069
```

## 2. Две ноды (локально)

```bash
# Терминал 1:
./waters-node --port 42069

# Терминал 2:
./waters-node --port 42070 --connect 127.0.0.1:42069
```

## 3. Три ноды (VPS + дача + дом)

```
┌─────────────────────────────────────────┐
│  VPS (сервер)                           │
│  Статический IP — мастер                │
└────────────┬────────────────────────────┘
             │ P2P sync
       ┌─────┴─────┐
       ▼           ▼
┌──────────┐ ┌──────────┐
│ Дача     │ │ Дом      │
│ Win:42070│ │ Ubu:42071│
│ динамич. │ │ динамич. │
│ IP       │ │ IP       │
└──────────┘ └──────────┘
```

**На VPS (мастер):**
```bash
./waters-node --port 42069
```

**На даче / доме:**
```bash
./waters-node --port 42070 --connect <VPS_IP>:42069
```

**Или вручную после запуска:**
```
/connect <VPS_IP>:42069
```

## 4. Проверка связи

```
status
# Должно быть:
#   Peers: 2
#   Node: waters-vps
#   Bridges: llm-deepseek, chat, duckduckgo
```

## 5. Групповой чат между нодами

```
/say 1 привет всем с VPS сервера
# Все ноды видят сообщение в Redis Stream

/chat 1 как слышно?
# Чат с показом последних сообщений

/opinions 1
# Последние сообщения группы
```

## 6. Проброс портов

Если нода за NAT:

```bash
# Linux: iptables или UPnP
# Windows: пробросить порт 42070 в роутере

# Альтернатива: использовать VPS как relay
# Все ноды подключаются к VPS, VPS видит всех
```

## 7. Топология

| Сценарий | Как соединять |
|----------|--------------|
| Локальный тест | `--connect 127.0.0.1:42069` |
| Две ноды в одной сети | mDNS автообнаружение |
| Через интернет | Клиенты → VPS со статическим IP |
| За NAT | Все → VPS (hub-and-spoke) |
| 6 нод в группе | Каждая соединяется со всеми |
