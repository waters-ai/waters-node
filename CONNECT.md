# Подключение нод (P2P)

## 1. Одиночная нода

```bash
./waters-node --port 42069
# → http://localhost:42069
```

## 2. Две ноды (локально)

```bash
# Терминал 1:
./waters-node --port 42069

# Терминал 2:
./waters-node --port 42070 --connect 127.0.0.1:42069
```

## 3. Три ноды (177 + дача + дом)

```
┌─────────────────────────────────────────┐
│  177 (сервер) 87.242.102.177:42069     │
│  Статический IP — мастер                │
└────────────┬────────────────────────────┘
             │ P2P sync
       ┌─────┴─────┐
       ▼           ▼
┌──────────┐ ┌──────────┐
│ Дача     │ │ Дом      │
│ Win:42070│ │ Ubu:42071│
│ динамич. │ │ динамич. │
│ IP       │ │ IP       │
└──────────┘ └──────────┘
```

**На 177 (мастер):**
```bash
./waters-node --port 42069
```

**На даче / доме:**
```bash
./waters-node --port 42070 --connect 87.242.102.177:42069
```

**Или вручную после запуска:**
```
/connect 87.242.102.177:42069
```

## 4. Проверка связи

```
status
# Должно быть:
#   Peers: 2
#   Node: waters-177
#   Bridges: llm-deepseek, chat, duckduckgo
```

## 5. Групповой чат между нодами

```
/say 1 привет всем с 177 сервера
# Все ноды видят сообщение в Redis Stream

/chat 1 как слышно?
# Чат с показом последних сообщений

/opinions 1
# Последние сообщения группы
```

## 6. Проброс портов

Если нода за NAT:

```bash
# Linux: iptables или UPnP
# Windows: пробросить порт 42070 в роутере

# Альтернатива: использовать 177 как relay
# Все ноды подключаются к 177, 177 видит всех
```

## 7. Топология

| Сценарий | Как соединять |
|----------|--------------|
| Локальный тест | `--connect 127.0.0.1:42069` |
| Две ноды в одной сети | mDNS автообнаружение |
| Через интернет | Клиенты → мастер со статическим IP |
| За NAT | Все → 177 (hub-and-spoke) |
| 6 нод в группе | Каждая соединяется со всеми |
