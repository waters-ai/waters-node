# Security & Privacy — WATERS Node

## Модель доступа: личное vs групповое

```
┌──────────────────────────────────────────────────┐
│                НОДА (ваша)                        │
│                                                    │
│  🔒 ЛИЧНОЕ (Personal)    │  👥 ГРУППОВОЕ (Shared) │
│                          │                         │
│  ● Ваши агенты           │  ● Агенты, которые вы   │
│  ● Ваши скилы            │    разрешили шарить    │
│  ● Ваши бриджи           │  ● Общий чат группы    │
│  ● Локальные файлы        │  ● Findings (результаты)│
│  ● Redis DB (ваш экземпляр)│  ● Общие ресурсы       │
└──────────────────────────┴─────────────────────────┘
```

### Что НЕ уходит из ноды

| Ресурс | Куда уходит | Контроль |
|--------|-------------|----------|
| API-ключи | ❌ Никуда | Локальный .env / redis |
| Локальные файлы | ❌ Никуда | Только через MCP-мосты |
| Персональные агенты | ❌ Никуда | Personal scope |
| Redis данные | ❌ Никуда | Локальный/удалённый Redis |
| SSH-соединения | ❌ Никуда | Локальный конфиг |

### Что можно шарить (и как)

| Ресурс | Команда | Требует подтверждения |
|--------|---------|----------------------|
| Skill | `/share skill <name>` | ✅ peer approval |
| Bridge | `/share bridge <name>` | ✅ peer approval |
| Agent | `/share agent <id>` | ✅ владелец агента |
| Chat | Автоматически в группе | ✅ по токену группы |
| Findings | Автоматически в группе | ✅ по режиму группы |

### Уровни видимости

```
personal  — только ваша нода (default)
group     — ноды в группе (по токену)
public    — все ноды в P2P-сети
```

## Команды безопасности

```
/privacy show            — показать политики доступа
/privacy share skill X   — разрешить группе использовать ваш скил
/privacy share bridge X  — разрешить группе использовать ваш бридж
/privacy revoke skill X  — отозвать доступ к скилу
/privacy personal        — всё в personal (по умолчанию)
```

## SecurityLearner — агент безопасности, который учится

Встроенный модуль `SecurityLearner` записывает все события безопасности и адаптирует политики на основе опыта.

### Как учится

| Событие | Что делает Learner |
|---------|-------------------|
| Peer успешно шарит > 10 раз | ✅ Auto-trust — повышает уровень доверия |
| Peer 3+ неудачных попыток | ⚠️ Предлагает заблокировать |
| 5+ аномалий за короткое время | 🚫 Auto-block без подтверждения |
| Новая нода с высоким риском | ❓ Unknown → требует approval |
| Частое использование бриджа | 📊 Адаптирует приоритет |

### Уровни доверия

```
Unknown  → Low   → Medium → High → Trusted
   ❓      ⚠️      🔶       🟢      ✅
```

- **Trusted**: auto-approve всего, без блокировок
- **High**: auto-approve для низкого риска
- **Medium**: требуется подтверждение
- **Low**: блокировка при риске > 3
- **Unknown**: блокировка при риске > 5

### Состояние

SecurityLearner сохраняет состояние в `.waters/security_learned.json`:
- Все события безопасности
- Доверие к каждому пиру
- Изученные правила (LearnedRule)
- Применяется авто-сохранение каждые 20 событий

## Плагины безопасности

Каждый плагин может перехватывать события и блокировать/разрешать:

| Плагин | Назначение |
|--------|-----------|
| logger | Логирует все события (аудит) |
| rate-limiter | Ограничивает вызовы LLM |
| yasa-screener | Проверяет команды на опасные паттерны |

Пример установки плагина:
```toml
# .waters/plugins/yasa-screener/plugin.toml
name = "yasa-screener"
version = "1.0.0"
description = "Проверяет команды на rm -rf, sudo, curl|bash"
hooks = ["agent_open", "bridge_call"]
```
