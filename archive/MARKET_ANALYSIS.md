## 3. Сильные стороны waters-node (что уже лучше)

### 3.1 P2P-сеть
Единственный агентный runtime с нативной P2P-сетью. Ни Hermes, ни SuperAGI, ни MS Agent Framework не имеют P2P — все они централизованные (клиент-сервер).

### 3.2 Компактность
11MB vs 200-500MB у конкурентов. Rust vs Python даёт 10-50x экономию памяти и ресурсов.

### 3.3 Автономия (DTN + L0-L4)
Для полевых условий, туризма, экспедиций — ни у кого нет. Hermes требует постоянного интернета.

### 3.4 Медиа-мосты
NDI, OBS, RTMP, HDMI — уникально для агентного runtime. Студии, блогеры, стримеры.

### 3.5 YASA-безопасность
Встроенная этическая система (8 аксиом, 5 заповедей) — ни у кого из конкурентов нет.

## 4. Слабые стороны (что нужно догонять)

### 4.1 Web UI / Dashboard ❓
**Вопрос пользователя: "нужна ли ноде веморда?"**

Hermes — красивый TUI + веб-дашборд.
SuperAGI — полноценный веб-интерфейс.
MS Agent Framework — Azure Studio.

У waters-node есть веб-дашборд (HTTP API + SSE), но:
- Нет drag-and-drop агентов
- Нет визуального редактора скиллов
- Нет мониторинга в реальном времени (графики)
- Нет управления группами через GUI

**Решение:** Легковесный React-дашборд (не отдельный бинарник), встроенный в ноду. Не как SuperAGI (500MB), а как встроенная страница.

### 4.2 Self-improving skills
Hermes делает это на Python с циклом: execute → review → improve.
У нас есть `skill_evolve.rs` (создан), но нет интеграции с LLM для авто-ревью.

### 4.3 Multi-platform gateway
Hermes: 22 платформы. У нас: Telegram + WhatsApp + WeChat (3).

**Нужно добавить:** Discord, Slack, Signal, Email.

### 4.4 Cross-session memory
Hermes: FTS5 + Honcho. У нас: только Redis (no search).

**Нужно:** Redisearch или SQLite FTS5 для поиска по старым сессиям.

### 4.5 Plugin system
Hermes: `ctx.llm` + `tool_override`. У нас: только BridgeProvider без плагинов.

### 4.6 Экосистема
Hermes: `huggingface/skills`, `agentskills.io`. У нас: только локальные скилы.

## 5. Нужен ли веб-дашборд?

### Аргументы ЗА

| Аспект | Почему |
|--------|--------|
| **UX для семьи** | Мама/папа не будут вводить `/chat создай задачу` |
| **Бизнес-клиенты** | Jira/1C/SAP интеграция через GUI, не через CLI |
| **Мониторинг** | Статус нод, нагрузка, ошибки — глазами, не `/status` |
| **Управление агентами** | Список агентов, их статус, рейтинг — визуально |
| **Конкуренты** | У всех есть (Hermes TUI, SuperAGI Web, MS Studio) |

### Аргументы ПРОТИВ

| Аспект | Почему |
|--------|--------|
| **11MB** | Если добавить React — бинарник вырастет |
| **LLM как UI** | Чат — интерфейс будущего. "Кнопки — это прошлое" |
| **P2P философия** | Каждая нода — терминал. GUI = централизация |
| **Компактность** | waters-node позиционируется как лёгкий. GUI утяжелит |

### Вердикт: Dashboard Light

**НЕ** нужен полноценный SuperAGI-дашборд. Но нужна **встроенная веб-страница** с:

1. **Чат** — уже есть (SSE)
2. **Статус нод** — список пиров, их LLM, нагрузка
3. **Агенты** — кто активен, рейтинг, скилы
4. **Голос** — кнопка записи (MediaRecorder → SSE)

Фреймворк: **Preact** (3KB, не React). Встраивается в бинарник через `include_str!`.
