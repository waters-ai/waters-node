## 2. Нефункциональные требования

| ID | Требование | Значение |
|----|-----------|---------|
| NF1 | Размер бинарника | ≤ 11MB |
| NF2 | Память (idle) | < 50MB RAM без активных агентов |
| NF3 | Задержка голоса (рация) | < 500ms |
| NF4 | Задержка голоса (агент) | < 5s |
| NF5 | Concurrent агентов | макс 10 на ноду |
| NF6 | Восстановление после сбоя | kill -9 → все данные целы |
| NF7 | Автономность | L0-L4: от полной сети до safe mode |
| NF8 | Персистентность | Redis — все данные на диске |
| NF9 | Масштабирование | до 6 нод в группе, до 60 агентов |
| NF10 | Язык | CLI/чат: русский, английский |
| NF11 | Обновление | замена бинарника, перезапуск |
| NF12 | Платформы | Linux x64, Windows x64 (macOS — план) |
| NF13 | Зависимости | только Redis 6+ (LLM — опционально) |

## 3. Требования безопасности

| ID | Требование | Механизм |
|----|-----------|---------|
| S1 | Досмотр агента перед запуском | YASA: проверка bridges, prompt, imported_from |
| S2 | Блокировка опасных команд | prompt scanning: rm -rf, sudo, curl\|bash |
| S3 | Проверка источника импорта | only trusted origins |
| S4 | Чат-аппрув входящих нод | запрос владельцу: "нода X просит доступ" |
| S5 | Чат-аппрув входящих агентов | запрос владельцу: "агент X хочет войти" |
| S6 | Heartbeat мониторинг | контроль пульса пилотов и агентов |
| S7 | Safe mode (L4) | 0 LLM, только маяк и базовая логика |
| S8 | Рейтинговый барьер | агенты с рейтингом < 0.3 блокируются |
| S9 | Personal bridges | мост виден только агентам своей ноды |
| S10 | Аудит | бортовой журнал всех действий агента |
| S11 | Concurrency cap | защита от перегрузки (макс 10) |
| S12 | Offline queue | события не теряются при обрыве связи |
